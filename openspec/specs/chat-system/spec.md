# 聊天系统规范

## 新增需求

### 需求：玩家可以发送文字消息
系统必须允许玩家在房间内发送简短文字消息，进行实时交流。

#### 场景：发送消息
- **当** 玩家在输入框输入"大家好"并点击发送
- **那么** 系统必须将消息广播给房间内所有玩家，显示在聊天区域

#### 场景：消息长度限制
- **当** 玩家输入超过 100 个字符
- **那么** 系统必须截断消息或提示"消息过长，最多 100 字符"

### 需求：玩家可以发送预设表情
系统必须提供预设表情（如点赞、大笑、哭泣），方便快速互动。

#### 场景：发送表情
- **当** 玩家点击表情面板中的"点赞"表情
- **那么** 系统必须发送表情，并在玩家头像旁显示 3 秒后消失

#### 场景：表情频率限制
- **当** 玩家在 10 秒内连续点击表情超过 5 次
- **那么** 系统必须提示"发送过快，请稍后再试"

### 需求：系统必须过滤敏感词
系统必须过滤聊天消息中的敏感词和不当内容。

#### 场景：敏感词替换
- **当** 玩家发送包含敏感词的消息
- **那么** 系统必须将敏感词替换为 `***` 后再广播

### 需求：玩家可以禁言其他玩家（仅房主）
系统必须允许房主禁言捣乱玩家，限制其发送消息。

#### 场景：禁言玩家
- **当** 房主点击玩家头像，选择"禁言"
- **那么** 系统必须禁止该玩家发送文字消息和表情，持续 5 分钟

#### 场景：解除禁言
- **当** 房主再次点击已被禁言的玩家，选择"解除禁言"
- **那么** 系统必须恢复该玩家的发言权限

### 需求：聊天记录必须限制数量
系统必须保留最近 50 条聊天记录，避免内存占用过高。

#### 场景：聊天记录滚动
- **当** 聊天记录超过 50 条
- **那么** 系统必须删除最早的记录，保持队列长度为 50
