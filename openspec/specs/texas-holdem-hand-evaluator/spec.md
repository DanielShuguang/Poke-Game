## 新增需求

### 需求：7选5最优牌型选取
系统必须从玩家 2 张底牌与最多 5 张公牌中，枚举所有 C(7,5)=21 种组合（River 时），评估每种组合的牌型等级，返回最高分组合作为该玩家最终牌型。River 前公牌不足5张时，从实际可用牌中枚举。

#### 场景：River 后最优选取
- **当** 5张公牌已全部翻出，玩家需摊牌
- **那么** 系统枚举21种5张组合，返回牌型等级最高的组合

#### 场景：公牌不足时选取
- **当** 公牌少于5张（仅用于即时胜率估算）
- **那么** 系统从底牌+现有公牌中枚举所有可能的5张组合

### 需求：全牌型识别
系统必须正确识别以下全部10种牌型，优先级从高到低：
1. 皇家同花顺（Royal Flush）
2. 同花顺（Straight Flush）
3. 四条（Four of a Kind）
4. 葫芦（Full House）
5. 同花（Flush）
6. 顺子（Straight）
7. 三条（Three of a Kind）
8. 两对（Two Pair）
9. 一对（One Pair）
10. 高牌（High Card）

#### 场景：识别皇家同花顺
- **当** 5张牌为同花色的 A-K-Q-J-10
- **那么** 系统返回牌型等级：皇家同花顺

#### 场景：识别同花顺
- **当** 5张牌花色相同且点数连续（非 A-K-Q-J-10）
- **那么** 系统返回牌型等级：同花顺，并以最高牌点数作为比较键

#### 场景：识别葫芦
- **当** 5张牌包含3张相同点数和2张相同点数
- **那么** 系统返回牌型等级：葫芦，三条点数优先比较

#### 场景：识别顺子含 A 低顺
- **当** 5张牌为 A-2-3-4-5（A 作为最低牌）
- **那么** 系统识别为顺子，以5为最高牌点数（而非 A）

### 需求：牌型大小比较
系统必须能比较两位或多位玩家的最优牌型，在牌型等级相同时依次比较踢脚牌（Kicker）点数，直至分出胜负或判定平局。

#### 场景：同等级牌型比较踢脚牌
- **当** 两位玩家均为一对，且对子点数相同
- **那么** 系统依次比较第1、第2、第3张踢脚牌，以第一张不同的踢脚牌决定胜负

#### 场景：完全相同牌型平局
- **当** 两位玩家最优5张牌完全相同（点数与花色无关）
- **那么** 系统判定平局，双方平分该底池

### 需求：牌型评分编码
系统必须将牌型评分编码为可直接比较的整数，格式为：`[牌型等级(4位)] [主要点数序列(20位)] [踢脚牌序列(20位)]`，高位整数代表更强牌型。

#### 场景：整数比较决定胜负
- **当** 系统比较两手牌
- **那么** 直接比较两个整数值，数值更大者获胜，相等则平局
