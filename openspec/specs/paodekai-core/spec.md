## 新增需求

### 需求：牌的表示
系统必须用 `PdkCard` 表示每张牌，包含点数（3–A、2、小王、大王）和花色（♠♥♣♦ 或无花色）。
点数枚举大小：3 < 4 < 5 < 6 < 7 < 8 < 9 < T < J < Q < K < A < 2 < 小王 < 大王。
同点数时花色：♠ > ♥ > ♣ > ♦。

#### 场景：单张比较
- **当** 比较两张单牌
- **那么** 点数大的牌胜出；点数相同时花色大的牌胜出

#### 场景：大小王识别
- **当** 牌的点数为 Joker
- **那么** 该牌无花色，且大王 > 小王

---

### 需求：牌型识别
系统必须能识别以下合法牌型：单张、对子、三张、顺子（≥5 张连续单张）、连对（≥3 对连续对子）、飞机（≥2 组连续三张）、炸弹（4 张相同点数）、王炸（大王+小王）。
2 和大小王禁止参与顺子/连对/飞机。

#### 场景：识别顺子
- **当** 玩家选择 5 张连续单牌且不含 2 或王
- **那么** 系统识别为顺子

#### 场景：拒绝含 2 的顺子
- **当** 玩家选择包含点数 2 的连续牌
- **那么** 系统拒绝，提示不合法

#### 场景：识别炸弹
- **当** 玩家选择 4 张相同点数的牌
- **那么** 系统识别为炸弹

#### 场景：识别王炸
- **当** 玩家选择大王和小王
- **那么** 系统识别为王炸

---

### 需求：牌型比较
系统必须按以下规则比较两手牌：相同牌型比较关键牌大小；王炸大于一切；炸弹大于非炸弹；炸弹之间比点数；顺子/连对/飞机必须等长才可比较。

#### 场景：炸弹压非炸弹
- **当** 上家出任意非炸弹牌型
- **那么** 炸弹可以跟出且必然胜过上家

#### 场景：顺子等长比较
- **当** 上家出 7 张顺子
- **那么** 只有 7 张顺子才能跟，5 张或 6 张顺子禁止跟

#### 场景：王炸最大
- **当** 上家出炸弹或任意牌型
- **那么** 王炸可以跟出且必然胜过上家

---

### 需求：游戏流程状态机
系统必须按照 waiting → dealing → playing → round_end → game_over 状态机管理游戏生命周期。
`PdkGameState` 必须包含：`players`、`currentPlayerIndex`、`lastPlayedCards`、`lastPlayedType`、`passCount`、`phase`。

#### 场景：发牌
- **当** 游戏开始
- **那么** 系统随机洗牌，每人分 18 张，持有 ♠3 的玩家 `currentPlayerIndex` 设为其索引

#### 场景：首轮出牌必须含 ♠3
- **当** 玩家出第一手牌
- **那么** 系统必须校验手牌中包含 ♠3，否则拒绝

#### 场景：所有人 pass 后新轮开始
- **当** `passCount == 2`（另外两人均 pass）
- **那么** `lastPlayedCards` 清空，`lastPlayedType` 重置，最后出牌者成为新一轮起手玩家

#### 场景：胜负判定
- **当** 某玩家手牌清空
- **那么** 游戏进入 game_over 阶段，按出完顺序记录名次（头游/二游/末游）

---

### 需求：积分结算
系统必须按名次计算积分：头游 +2 分，二游 0 分，末游 -2 分。

#### 场景：三人结算
- **当** game_over 阶段触发
- **那么** 显示三人名次和积分变化（+2 / 0 / -2）
