# 房间管理规范

## 新增需求

### 需求：用户可以创建游戏房间
系统必须允许用户创建新房间，设置房间名称、游戏类型、最大人数等参数。

#### 场景：创建斗地主房间
- **当** 用户选择"创建房间"，选择游戏类型"斗地主"，输入房间名称"欢乐对战"
- **那么** 系统必须创建房间，固定人数为 3 人，并自动成为房主和玩家

#### 场景：创建自定义人数房间
- **当** 用户选择游戏类型"德州扑克"，设置人数范围为 2-6 人
- **那么** 系统必须创建房间，允许 2-6 人加入，房主自动成为玩家

### 需求：房主可以销毁房间
系统必须允许房主主动销毁房间，并通知所有玩家。

#### 场景：房主销毁房间
- **当** 房主点击"关闭房间"按钮
- **那么** 系统必须断开所有玩家连接，通知玩家"房间已关闭"，并释放资源

#### 场景：房主异常退出
- **当** 房主应用崩溃或网络中断超过 60 秒
- **那么** 系统必须自动解散房间，通知所有玩家"房主已离线，房间解散"

### 需求：用户可以加入房间
系统必须允许用户通过房间列表或手动输入 IP 加入房间。

#### 场景：通过房间列表加入
- **当** 用户在房间列表中点击房间"欢乐对战"
- **那么** 系统必须向房主发起连接请求，成功后进入等待大厅

#### 场景：房间已满
- **当** 用户尝试加入已满员房间
- **那么** 系统必须拒绝加入并提示"房间已满"

#### 场景：房间不存在
- **当** 用户手动输入 IP 地址连接不存在的房间
- **那么** 系统必须在 5 秒后提示"连接失败，请检查地址"

### 需求：玩家可以退出房间
系统必须允许玩家主动退出房间，保留其他玩家继续游戏。

#### 场景：游戏中退出
- **当** 玩家在游戏进行中点击"退出"按钮
- **那么** 系统必须提示"退出将视为认输，是否确认？"

#### 场景：等待中退出
- **当** 玩家在等待大厅中点击"退出"按钮
- **那么** 系统必须直接退出房间，通知房主更新玩家列表

### 需求：房主可以踢出玩家
系统必须允许房主在游戏开始前踢出玩家。

#### 场景：踢出玩家
- **当** 房主在玩家列表中点击"踢出"按钮
- **那么** 系统必须将该玩家移出房间，并通知被踢玩家"你已被移出房间"

### 需求：房间状态必须实时同步
系统必须确保所有玩家看到的房间状态（人数、玩家列表）保持一致。

#### 场景：玩家加入同步
- **当** 新玩家加入房间
- **那么** 系统必须在 1 秒内向所有玩家广播更新后的玩家列表

#### 场景：玩家退出同步
- **当** 玩家退出房间
- **那么** 系统必须在 1 秒内向剩余玩家广播更新后的玩家列表

## 修改需求

### 需求：用户可以创建游戏房间
系统必须允许用户创建新房间，设置房间名称、游戏类型、最大人数等参数。

#### 场景：创建斗地主房间
- **当** 用户选择"创建房间"，选择游戏类型"斗地主"，输入房间名称"欢乐对战"
- **那么** 系统必须创建房间，固定人数为 3 人，并自动成为房主和玩家

#### 场景：创建自定义人数房间
- **当** 用户选择游戏类型"德州扑克"，设置人数范围为 2-6 人
- **那么** 系统必须创建房间，允许 2-6 人加入，房主自动成为玩家

#### 场景：创建 21 点房间
- **当** 用户选择游戏类型"21 点"，设置人数为 2-7 人
- **那么** 系统必须创建房间，房主自动担任庄家角色，等待 1-6 名玩家加入

## 新增需求

### 需求：系统必须支持 blackjack 游戏类型标识
房间的 `gameType` 字段必须接受 `blackjack` 作为合法值，路由和房间过滤逻辑必须处理该类型。

#### 场景：按游戏类型过滤房间列表
- **当** 玩家在扫描页携带参数 `gameType=blackjack`
- **那么** 系统必须仅显示 `gameType` 为 `blackjack` 的房间

#### 场景：大厅开始游戏路由至 21 点页面
- **当** 21 点房间所有玩家准备完成，房主点击"开始游戏"
- **那么** `RoomLobbyPage._navigateToGame()` 必须创建 `BlackjackNetworkAdapter` 并跳转至 `/blackjack`
