## 新增需求

### 需求：横屏布局锁定
跑得快游戏页面必须在 `initState` 时锁定横屏方向，在 `dispose` 时恢复默认方向，与现有其他游戏页面保持一致。

#### 场景：进入游戏锁横屏
- **当** 玩家进入跑得快游戏页面
- **那么** 设备强制切换为横屏，不允许旋转

#### 场景：退出游戏恢复方向
- **当** 玩家退出游戏页面
- **那么** 屏幕方向恢复为系统默认

---

### 需求：三方玩家座位布局
游戏页面必须显示三个玩家区域：本地玩家在底部中央，对手玩家分列左上和右上。
每个玩家区域必须显示：头像、昵称、剩余手牌数量。

#### 场景：本地玩家区
- **当** 游戏进行中
- **那么** 底部区域显示本地玩家的手牌（可点击选择），以及剩余手牌数

#### 场景：对手玩家区
- **当** 游戏进行中
- **那么** 左上和右上分别显示对手的背面手牌扇形和剩余张数

---

### 需求：手牌选择与出牌操作
玩家必须能通过点击或拖拽选中手牌；选中后点击"出牌"按钮提交；当没有合法出牌时"出牌"按钮禁用，仅显示"不出"按钮；操作区必须包含"提示"按钮，点击后自动高亮最小合法出法手牌。

#### 场景：点击选牌
- **当** 玩家点击一张手牌
- **那么** 该牌上浮高亮显示为"已选中"状态；再次点击取消选中

#### 场景：拖拽滑选
- **当** 玩家手指滑过多张手牌
- **那么** 依次将经过的牌加入拖拽列表，实时预览最小合法牌组高亮；松手后确认选中

#### 场景：出牌合法性校验
- **当** 玩家点击"出牌"按钮
- **那么** 系统校验选中牌是否构成合法牌型且大于上家；不合法时按钮禁用无响应

#### 场景：起手方无"不出"选项
- **当** 玩家是新一轮起手方
- **那么** "不出"按钮隐藏，玩家必须出牌

#### 场景：点击提示按钮
- **当** 玩家点击"提示"按钮（仅本玩家回合显示）
- **那么** 自动高亮最小合法出法手牌；若无合法出法则按钮禁用

---

### 需求：出牌区展示
中央出牌区必须显示最近一手出牌及出牌玩家名称。所有人 pass 后出牌区清空。

#### 场景：展示最新出牌
- **当** 任意玩家完成出牌
- **那么** 中央区域更新显示该玩家出的牌（带动画）和玩家名称

#### 场景：出牌区清空
- **当** `passCount == 2` 新轮开始
- **那么** 中央出牌区淡出清空

---

### 需求：退出确认
游戏页面左上角必须有退出按钮（`Icons.arrow_back`，`white70`），点击后弹出确认对话框，确认后返回上一页。

#### 场景：退出确认弹窗
- **当** 玩家点击退出按钮
- **那么** 弹出对话框询问"确认退出游戏？"；点击确认后 `Navigator.pop(context)`；点击取消关闭弹窗

---

### 需求：游戏结算弹窗
游戏结束时必须弹出结算弹窗，显示三人名次、头像和积分变化（+2 / 0 / -2），提供"再来一局"和"返回首页"按钮。

#### 场景：胜利结算
- **当** 本地玩家最先出完手牌
- **那么** 弹出结算弹窗显示"头游！+2 分"及其他玩家排名

#### 场景：失败结算
- **当** 本地玩家最后出完手牌
- **那么** 弹出结算弹窗显示"末游 -2 分"

---

### 需求：主题风格一致性
跑得快页面必须使用与现有游戏一致的深色桌面背景色（`bgTable`）、深色渐变卡牌样式，退出按钮使用 `GameBackButton` 组件。

#### 场景：背景色
- **当** 进入游戏页面
- **那么** 页面背景使用 `Theme.of(context).extension<GameColors>()!.bgTable`

#### 场景：卡牌样式
- **当** 展示手牌
- **那么** 卡牌使用统一的深色渐变+发光边框样式组件
