# 局域网发现规范

## 新增需求

### 需求：设备可以自动发现局域网内的游戏房间
系统必须允许设备在局域网内自动扫描并发现可用的游戏房间，无需手动输入地址。

#### 场景：Android 设备自动发现房间
- **当** 用户进入"局域网对战"页面
- **那么** 系统自动扫描局域网，在 5 秒内显示所有可用房间列表

#### 场景：iOS 设备通过 mDNS 发现房间
- **当** 用户使用 iOS 设备进入"局域网对战"页面
- **那么** 系统使用 NSD (Network Service Discovery) 发现服务类型为 `_pokegame._tcp` 的房间

#### 场景：手动输入 IP 地址
- **当** 自动发现失败或用户选择手动模式
- **那么** 系统必须允许用户输入房主 IP 地址（如 192.168.1.100:8080）

### 需求：房间信息必须包含完整元数据
系统必须确保每个房间广播的信息包含以下字段：
- 房间 ID（UUID）
- 房间名称
- 游戏类型（如斗地主、德州扑克）
- 当前人数/最大人数
- 房主设备名称
- 网络地址（IP + 端口）

#### 场景：UDP 广播房间信息
- **当** 房主创建房间
- **那么** 系统每 2 秒通过 UDP 端口 8081 广播房间信息 JSON

#### 场景：mDNS 服务注册
- **当** iOS/macOS 设备创建房间
- **那么** 系统注册 NSD 服务，服务名为房间名称，类型为 `_pokegame._tcp`

### 需求：设备可以停止扫描和广播
系统必须允许设备在离开发现页面或关闭房间时停止扫描或广播，释放网络资源。

#### 场景：停止扫描
- **当** 用户退出"局域网对战"页面
- **那么** 系统必须停止所有 UDP 监听或 NSD 浏览操作

#### 场景：停止广播
- **当** 房主关闭房间或应用进入后台
- **那么** 系统必须停止 UDP 广播或注销 NSD 服务

### 需求：系统必须检测网络环境问题
系统必须检测并提示用户网络环境问题（如 WiFi 未连接、AP 隔离）。

#### 场景：检测 WiFi 连接状态
- **当** 用户尝试进入"局域网对战"页面
- **那么** 系统必须检测设备是否连接 WiFi，若未连接则提示用户

#### 场景：检测 AP 隔离
- **当** 扫描超过 10 秒未发现任何房间
- **那么** 系统必须提示用户"可能存在 WiFi 隔离，请检查路由器设置"
