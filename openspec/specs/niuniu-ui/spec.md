## 新增需求

### 需求：系统必须提供牛牛游戏页面
系统必须提供 `NiuniuPage`（`ConsumerStatefulWidget`），接收 `isOnline: bool` 和 `networkAdapter: NiuniuNetworkAdapter?` 参数，在 `initState` 中锁定横屏方向。

#### 场景：单机模式初始化
- **当** `NiuniuPage(isOnline: false)` 启动
- **那么** 页面必须初始化一局含1名人类玩家（ID='human'）和3名AI玩家的游戏，直接进入下注阶段

#### 场景：联机模式初始化
- **当** `NiuniuPage(isOnline: true, networkAdapter: adapter)` 启动
- **那么** 页面必须等待适配器同步状态，由 Host 驱动游戏进程

### 需求：系统必须展示下注界面
下注阶段必须显示筹码面额按钮（10/50/100/500）、确认下注按钮和当前筹码余额。

#### 场景：玩家下注
- **当** 玩家点击面额按钮后点击"确认下注"
- **那么** 系统必须扣除筹码，记录下注额，等待其他玩家下注完成

#### 场景：筹码不足
- **当** 玩家筹码少于所选面额
- **那么** 对应面额按钮必须被禁用

### 需求：系统必须展示牌桌与手牌
showdown 阶段必须展示庄家与所有闲家的5张牌，并在每张牌下方显示牌型名称（无牛/牛X/牛牛/五小牛/炸弹）。

#### 场景：翻牌展示
- **当** 进入 showdown 阶段
- **那么** 庄家手牌必须完整展示；闲家依次翻开，每次翻牌间隔400ms（单机模式）

#### 场景：显示倍率标签
- **当** 手牌翻开后
- **那么** 每位玩家手牌下方必须显示牌型名称和倍率（如"牛牛 ×3"）

### 需求：系统必须展示结算覆盖层
结算阶段必须以覆盖层形式展示每位玩家的输赢金额，3秒后显示"再来一局"按钮。

#### 场景：结算显示
- **当** 进入 settlement 阶段
- **那么** 覆盖层必须对每位玩家显示"+ X" 或 "- X"的筹码变化，赢方显示绿色，输方显示红色

#### 场景：再来一局
- **当** 用户点击"再来一局"
- **那么** 系统必须重置游戏状态并返回下注阶段

### 需求：系统必须在非本地玩家行动时禁用操作按钮
联机模式下，当轮到其他玩家操作时，本地玩家的下注按钮必须禁用，并显示"等待其他玩家下注..."提示文本。

#### 场景：等待提示
- **当** 联机模式且当前轮到其他玩家下注
- **那么** 操作区必须显示"等待其他玩家下注..."并禁用所有下注按钮
