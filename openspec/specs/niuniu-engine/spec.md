## 新增需求

### 需求：系统必须能表示牛牛手牌并计算牌型
系统必须提供 `NiuniuHand` 实体，能从5张牌中枚举所有3张组合，判断是否有牛，计算牌型等级与比较倍率。

#### 场景：判断无牛
- **当** 5张牌中任意3张之和均不能被10整除
- **那么** `NiuniuHand.rank` 必须返回 `NiuniuRank.noPoints`，倍率为1

#### 场景：判断牛6
- **当** 存在3张之和为10的倍数，且余2张之和 mod 10 == 6
- **那么** `NiuniuHand.rank` 必须返回 `NiuniuRank.niu6`，倍率为1

#### 场景：判断牛7
- **当** 余2张之和 mod 10 == 7
- **那么** 倍率必须为2

#### 场景：判断牛牛
- **当** 余2张之和 mod 10 == 0
- **那么** `NiuniuHand.rank` 必须返回 `NiuniuRank.niuNiu`，倍率为3

#### 场景：判断五小牛
- **当** 5张牌全部点数 ≤5（A/2/3/4/5）且点数之和 ≤10
- **那么** `NiuniuHand.rank` 必须返回 `NiuniuRank.fiveSmall`，倍率为5

#### 场景：判断炸弹
- **当** 5张牌中有4张同点数
- **那么** `NiuniuHand.rank` 必须返回 `NiuniuRank.bomb`，倍率为5，优先级高于五小牛

### 需求：系统必须能比较两手牌大小
系统必须提供 `NiuniuHand.compareTo(other)` 方法，按牌型等级 → 最高单张点数的规则比较两手牌。

#### 场景：牛牛 vs 牛9
- **当** 一手牌为牛牛，另一手为牛9
- **那么** 牛牛必须胜出

#### 场景：同为牛6比大小
- **当** 两手牌均为牛6
- **那么** 系统必须比较5张牌中最大单张点数（K > Q > J > 10 > ... > A）

### 需求：系统必须提供发牌 UseCase
系统必须提供 `DealNiuniuUseCase`，使用6副标准牌（312张）洗牌后为每位玩家发5张牌。

#### 场景：正常发牌
- **当** 调用 `DealNiuniuUseCase().call(state)` 且牌堆充足
- **那么** 每位玩家必须获得恰好5张牌，游戏阶段切换为 `showdown`

### 需求：系统必须提供结算 UseCase
系统必须提供 `SettleNiuniuUseCase`，将每位闲家手牌与庄家比较，按倍率结算筹码。

#### 场景：闲家牛牛胜庄家牛6
- **当** 闲家为牛牛(×3)，庄家为牛6(×1)
- **那么** 闲家净赢 `bet × 3`，庄家净输 `bet × 3`

#### 场景：庄家胜闲家
- **当** 庄家手牌优于闲家
- **那么** 庄家净赢 `bet × 闲家倍率`，闲家净输相同金额

#### 场景：平局
- **当** 庄家与闲家牌型、最大牌相同
- **那么** 庄家胜（庄家优先），闲家净输 `bet × 1`

### 需求：系统必须提供 NiuniuGameState 实体
系统必须提供使用 `freezed` 的不可变 `NiuniuGameState`，包含牌堆、庄家ID、玩家列表、当前阶段枚举（betting/dealing/showdown/settlement）、以及 `toJson`/`fromJson`（含 `includeAllCards` 参数）。

#### 场景：手牌隐私序列化
- **当** `toJson(includeAllCards: false)` 在 `showdown` 前被调用
- **那么** 非本地玩家的手牌列表必须为空

#### 场景：结算后序列化
- **当** 阶段为 `settlement`
- **那么** 所有玩家手牌必须包含在序列化结果中
