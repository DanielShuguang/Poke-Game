# 升级局域网联机规范

## 新增需求

### 需求：系统必须实现升级网络适配器
系统必须提供 `ShengjiNetworkAdapter` 处理局域网通信。

#### 场景：适配器初始化
- **当** 游戏开始
- **那么** 系统必须创建 `ShengjiNetworkAdapter`，传入 incomingStream、broadcastFn、notifier、isHost、localPlayerId

#### 场景：Host 接收行动
- **当** Host 收到 Client 发送的行动消息
- **那么** 适配器必须解析消息并调用 notifier 执行对应操作

#### 场景：Client 发送行动
- **当** Client 玩家执行叫牌或出牌
- **那么** 适配器必须封装行动消息并发送给 Host

### 需求：系统必须定义网络行动类型
系统必须定义升级游戏专用的网络行动消息格式。

#### 场景：叫牌行动
- **当** 玩家叫牌
- **那么** 系统必须发送 `CallTrumpAction`，包含 `playerId`、`callType`（pair/tractor/joker）、`suit`、`rank`

#### 场景：出牌行动
- **当** 玩家出牌
- **那么** 系统必须发送 `PlayCardsAction`，包含 `playerId`、`cards`（牌列表）

#### 场景：状态同步
- **当** Host 执行行动后
- **那么** 系统必须广播 `StateSyncAction`，包含完整游戏状态 JSON

### 需求：系统必须保护手牌隐私
系统必须确保 Client 端仅能看到自己的手牌。

#### 场景：状态同步隐藏手牌
- **当** Host 广播游戏状态
- **那么** 必须调用 `toJson(includeAllCards: false)`，隐藏非本地玩家手牌

#### 场景：Client 接收状态
- **当** Client 收到状态同步消息
- **那么** 系统必须仅恢复 `localPlayerId` 对应的手牌，其他玩家手牌置空

### 需求：系统必须实现超时托管
系统必须在玩家超时后代为执行最小操作。

#### 场景：叫牌超时
- **当** 玩家 35 秒内未叫牌
- **那么** Host 必须自动执行"不叫"操作

#### 场景：出牌超时
- **当** 玩家 35 秒内未出牌
- **那么** Host 必须自动出最小合法牌

#### 场景：超时通知
- **当** 超时托管执行
- **那么** 系统必须通知所有玩家"玩家 X 超时，系统代为操作"

### 需求：系统必须处理网络异常
系统必须处理断线重连等网络异常情况。

#### 场景：玩家断线
- **当** 玩家网络中断
- **那么** 系统必须保留玩家状态，由 AI 代打

#### 场景：重连恢复
- **当** 玩家重新连接
- **那么** 系统必须同步当前游戏状态，恢复玩家控制

### 需求：系统必须序列化游戏状态
系统必须支持游戏状态的 JSON 序列化和反序列化。

#### 场景：状态序列化
- **当** 需要同步游戏状态
- **那么** 系统必须将 `ShengjiGameState` 序列化为 JSON，包含 phase、players、teams、trumpInfo、scores 等字段

#### 场景：状态反序列化
- **当** 收到状态同步消息
- **那么** 系统必须从 JSON 恢复 `ShengjiGameState` 对象
