## 新增需求

### 需求：游戏页面必须展示牌桌布局
页面必须分为庄家区（上方）和玩家区（下方），各区展示手牌、当前点数。联机模式下多名玩家横向排列。

#### 场景：单机布局
- **当** 进入单机模式
- **那么** 页面必须显示：顶部庄家区（含暗牌）、底部当前玩家区、中间操作按钮栏

#### 场景：联机多玩家布局
- **当** 进入联机模式且玩家数 ≥ 2
- **那么** 页面必须在中部水平滚动显示所有玩家区域，高亮当前行动玩家

### 需求：操作按钮必须根据当前状态动态启用/禁用
系统必须根据手牌张数和游戏阶段控制按钮可用性。

#### 场景：初始两张牌时全部按钮可用
- **当** 玩家恰好持有两张牌且为己方回合
- **那么** Hit / Stand / Double / Split（仅同点时）/ Surrender 必须全部显示

#### 场景：摸牌后限制操作
- **当** 玩家手牌超过两张
- **那么** 系统必须仅显示 Hit 和 Stand，隐藏 Double / Split / Surrender

#### 场景：非己方回合禁用操作
- **当** 不是本玩家回合（联机模式或等待庄家）
- **那么** 所有操作按钮必须禁用，显示"等待其他玩家操作..."提示

### 需求：下注界面必须在每局开始前展示
每局开始前，系统必须展示下注 UI，玩家选择筹码面额点击牌桌区域完成下注，确认后不可更改。

#### 场景：下注确认
- **当** 玩家点击"确认下注"
- **那么** 系统必须锁定下注金额，开始发牌动画

#### 场景：下注不足
- **当** 玩家未下注就点击确认
- **那么** 系统必须提示"请先下注"，禁止开始游戏

### 需求：牌面动画必须平滑展示发牌过程
发牌时必须有从牌堆飞入的动画（≤300ms/张），翻牌时必须有翻转动画（≤200ms）。

#### 场景：发牌动画
- **当** 游戏开始发牌
- **那么** 每张牌必须按顺序以动画形式飞入对应玩家区域，完成后显示点数

#### 场景：庄家暗牌翻开动画
- **当** 庄家暗牌翻开
- **那么** 系统必须播放 180° 翻转动画后显示真实牌面

### 需求：结算界面必须清晰展示每手牌结果
游戏结束后必须显示结算覆盖层，展示每位玩家的手牌、点数、胜负状态和筹码变化。

#### 场景：结算展示
- **当** 庄家行动结束，触发结算
- **那么** 系统必须显示结算卡片：赢/输/平/Blackjack，筹码 +/- 数量，保持 3 秒后显示"再来一局"按钮
