# 玩家管理规范

## 新增需求

### 需求：系统必须为每位玩家分配唯一标识
系统必须在玩家加入房间时分配唯一的玩家 ID 和显示名称。

#### 场景：自动生成玩家名称
- **当** 玩家首次加入房间
- **那么** 系统必须生成默认名称"玩家 X"（X 为递增数字），并分配 UUID

#### 场景：自定义玩家名称
- **当** 玩家在设置中修改名称为"高手"
- **那么** 系统必须更新玩家名称，并在房间内实时同步

### 需求：玩家可以标记准备状态
系统必须允许玩家在等待大厅标记"已准备"或"取消准备"。

#### 场景：标记准备
- **当** 玩家点击"准备"按钮
- **那么** 系统必须更新玩家状态为"已准备"，并广播给所有玩家

#### 场景：取消准备
- **当** 已准备的玩家点击"取消准备"按钮
- **那么** 系统必须更新玩家状态为"未准备"，并广播给所有玩家

### 需求：房主可以强制开始游戏
系统必须允许房主在满足最低人数时强制开始游戏。

#### 场景：所有玩家已准备
- **当** 房间内所有玩家都标记为"已准备"
- **那么** 系统必须自动开始游戏

#### 场景：房主强制开始
- **当** 房间人数达到最低要求（如斗地主 3 人），部分玩家未准备
- **那么** 房主可以点击"强制开始"按钮，系统必须踢出未准备玩家并开始游戏

### 需求：系统必须管理玩家座位
系统必须为每位玩家分配座位编号，并在游戏中维护座位顺序。

#### 场景：分配座位
- **当** 玩家加入房间
- **那么** 系统必须按加入顺序分配座位编号（1, 2, 3...）

#### 场景：玩家退出重排座位
- **当** 座位 2 的玩家退出房间
- **那么** 系统必须重新分配座位编号，保持连续性（原座位 3 变为 2）

### 需求：系统必须处理玩家离线
系统必须检测玩家网络状态，处理离线重连逻辑。

#### 场景：短暂离线重连
- **当** 玩家网络中断，在 60 秒内重新连接
- **那么** 系统必须恢复玩家状态，同步当前游戏进度

#### 场景：长时间离线
- **当** 玩家网络中断超过 60 秒
- **那么** 系统必须将玩家标记为"已离线"，若游戏中则视为认输

### 需求：玩家可以查看其他玩家信息
系统必须允许玩家在游戏中查看其他玩家的基本信息。

#### 场景：查看玩家信息
- **当** 玩家点击其他玩家头像
- **那么** 系统必须显示该玩家的名称、座位号、准备状态
