## 新增需求

### 需求：共享拖拽选牌组件
`DragSelectCardHand` 组件必须封装完整的拖拽选牌交互流程，包括指针事件监听、位置缓存、命中检测、预览高亮和滚动互斥，通过回调接口支持各游戏差异化的验证逻辑。

#### 场景：拖拽跨越多张牌
- **当** 玩家手指按下后横向滑过多张牌，且移动距离超过 `dragThreshold`
- **那么** 滑过的每张牌依次加入拖拽索引列表，调用 `calculatePreview` 回调计算预览索引，对应牌以预览状态显示

#### 场景：点击选中单张（未超过阈值）
- **当** 玩家手指按下后抬起，移动距离不超过 `dragThreshold`
- **那么** 不触发拖拽逻辑，由 `onTap(index)` 回调处理单击事件

#### 场景：拖拽结束确认选择
- **当** 玩家抬起手指结束拖拽
- **那么** 若 `calculatePreview` 返回的预览索引集合非空，调用 `onDragEnd(previewIndices)` 通知父组件；清除所有拖拽临时状态

#### 场景：拖拽取消
- **当** 拖拽被系统取消（来电、弹窗等 `PointerCancelEvent`）
- **那么** 清除拖拽状态，不调用 `onDragEnd`，不改变已有选中状态

#### 场景：禁用状态下不响应拖拽
- **当** `enabled` 为 `false`（非本玩家回合）
- **那么** 所有指针事件不响应，手牌不可拖拽也不可点击

### 需求：拖拽与滚动互斥
拖拽进行时必须禁止 `SingleChildScrollView` 的横向滚动，避免选牌手势与滚动冲突。

#### 场景：拖拽期间滚动冻结
- **当** 拖拽已激活（移动超过阈值）
- **那么** `SingleChildScrollView` 的 `physics` 切换为 `NeverScrollableScrollPhysics`

#### 场景：拖拽结束后滚动恢复
- **当** 拖拽结束或取消
- **那么** `SingleChildScrollView` 恢复默认滚动行为

### 需求：位置缓存与命中检测
组件必须通过标记 Widget 和 Element 树遍历缓存每张牌的 `Rect` 位置，支持重叠卡牌的精准命中检测。

#### 场景：重叠卡牌命中优先级
- **当** 指针位置同时落在多张重叠牌的 Rect 范围内
- **那么** 必须选中索引最大的牌（视觉上在最上层的牌）

#### 场景：位置缓存更新时机
- **当** 组件 build 完成后，或指针按下时
- **那么** 必须通过 `addPostFrameCallback` 重新遍历 Element 树更新 `Map<int, Rect>` 缓存

### 需求：可配置拖拽阈值
组件必须支持 `dragThreshold` 参数控制触发拖拽的最小移动距离。

#### 场景：零阈值立即拖拽
- **当** `dragThreshold` 设为 `0`
- **那么** 指针按下即标记为拖拽开始，首张牌立即加入拖拽列表

#### 场景：正阈值延迟拖拽
- **当** `dragThreshold` 设为正值（如 `4.0`）
- **那么** 指针移动距离未超过阈值时不触发拖拽，超过后才开始收集卡牌

### 需求：卡牌渲染委托
组件必须通过 `cardBuilder` 回调委托卡牌渲染，不内置任何游戏特定的卡牌样式。

#### 场景：传递拖拽状态给卡牌
- **当** 渲染每张卡牌时
- **那么** `cardBuilder(index, isDragged: bool, isPreview: bool)` 必须传递该牌的拖拽状态和预览状态，由各游戏决定样式
