## 修改需求

### 需求：用户可以创建游戏房间
系统必须允许用户创建新房间，设置房间名称、游戏类型、最大人数等参数。

#### 场景：创建斗地主房间
- **当** 用户选择"创建房间"，选择游戏类型"斗地主"，输入房间名称"欢乐对战"
- **那么** 系统必须创建房间，固定人数为 3 人，并自动成为房主和玩家

#### 场景：创建自定义人数房间
- **当** 用户选择游戏类型"德州扑克"，设置人数范围为 2-6 人
- **那么** 系统必须创建房间，允许 2-6 人加入，房主自动成为玩家

#### 场景：创建 21 点房间
- **当** 用户选择游戏类型"21 点"，设置人数为 2-7 人
- **那么** 系统必须创建房间，房主自动担任庄家角色，等待 1-6 名玩家加入

## 新增需求

### 需求：系统必须支持 blackjack 游戏类型标识
房间的 `gameType` 字段必须接受 `blackjack` 作为合法值，路由和房间过滤逻辑必须处理该类型。

#### 场景：按游戏类型过滤房间列表
- **当** 玩家在扫描页携带参数 `gameType=blackjack`
- **那么** 系统必须仅显示 `gameType` 为 `blackjack` 的房间

#### 场景：大厅开始游戏路由至 21 点页面
- **当** 21 点房间所有玩家准备完成，房主点击"开始游戏"
- **那么** `RoomLobbyPage._navigateToGame()` 必须创建 `BlackjackNetworkAdapter` 并跳转至 `/blackjack`
