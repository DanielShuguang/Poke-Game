## 新增需求

### 需求：系统必须正确计算手牌点数
系统必须将 2–10 计为面值，J/Q/K 计为 10，A 计为 1 或 11（取使手牌不爆的最大值）。超过 21 点为爆牌（Bust）。

#### 场景：A 自动切换为 1
- **当** 玩家手牌为 [A, 8]（19 点），再摸一张 5
- **那么** 系统必须将 A 计为 1，显示手牌点数 14，不判定为爆牌

#### 场景：Blackjack 识别
- **当** 玩家前两张牌为 [A, K]
- **那么** 系统必须标记该手牌为 Blackjack（21 点自然牌），赔率 1.5 倍

#### 场景：爆牌判定
- **当** 玩家手牌点数超过 21
- **那么** 系统必须立即将该手牌标记为 Bust，结束该手牌行动

### 需求：系统必须支持标准玩家操作
玩家在行动阶段必须能执行：Hit（摸牌）、Stand（停牌）、Double（加倍，仅限前两张牌）、Split（分牌，仅限两张相同点数的牌）、Surrender（投降，仅限前两张牌）。

#### 场景：Hit
- **当** 玩家选择 Hit
- **那么** 系统必须从牌堆顶发一张牌给玩家，重新计算点数

#### 场景：Stand
- **当** 玩家选择 Stand
- **那么** 系统必须结束该手牌行动，轮到下一位玩家或庄家

#### 场景：Double
- **当** 玩家手牌恰好为两张且选择 Double
- **那么** 系统必须将下注翻倍，发一张牌后自动 Stand

#### 场景：Split
- **当** 玩家手牌为两张相同点数的牌且选择 Split
- **那么** 系统必须将两张牌分为两手，各补一张牌，玩家依次对每手操作；Split 最多执行一次

#### 场景：Surrender
- **当** 玩家手牌恰好为两张且选择 Surrender
- **那么** 系统必须返还玩家一半下注，该手牌立即结束

### 需求：系统必须按轮次管理玩家行动顺序
联机模式下，系统必须按座位顺序逐一给每位玩家行动机会（Split 产生的第二手牌紧接第一手牌之后）。所有玩家完成后，庄家最后行动。

#### 场景：轮次推进
- **当** 当前玩家 Stand 或 Bust 或 Surrender
- **那么** 系统必须将 `currentPlayerIndex` 推进到下一位未完成的玩家

#### 场景：所有玩家完成后庄家行动
- **当** 所有玩家手牌均已完成（Stand / Bust / Surrender）
- **那么** 系统必须进入庄家行动阶段，执行 Hard 17 规则直至庄家 Stand 或 Bust

### 需求：系统必须正确结算胜负
系统必须在庄家完成行动后，对每位玩家的每手牌独立结算。

#### 场景：玩家点数 > 庄家且不爆
- **当** 玩家 22 点（Bust）— 不适用；玩家 19 点，庄家 17 点
- **那么** 系统必须判定玩家胜，返还 2 倍下注

#### 场景：庄家爆牌
- **当** 庄家点数超过 21
- **那么** 系统必须判定所有未爆玩家均胜，返还 2 倍下注

#### 场景：平局（Push）
- **当** 玩家与庄家点数相同（均非 Blackjack）
- **那么** 系统必须返还玩家原始下注，不额外获利

#### 场景：Blackjack 赔率
- **当** 玩家持有 Blackjack 而庄家非 Blackjack
- **那么** 系统必须返还 2.5 倍下注（原注 + 1.5 倍赢利）

### 需求：系统必须支持可选的五小龙规则
当 `GameConfig.fiveCardCharlie` 为 true 时，玩家持有 5 张牌且不爆必须自动赢（Surrender 除外）。

#### 场景：五小龙触发
- **当** 玩家摸第 5 张牌后点数 ≤ 21 且 `fiveCardCharlie` 为 true
- **那么** 系统必须立即标记该手牌为五小龙胜利，等同普通胜利赔率
