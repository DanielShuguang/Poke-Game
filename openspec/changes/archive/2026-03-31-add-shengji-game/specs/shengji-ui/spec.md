# 升级游戏界面规范

## 新增需求

### 需求：系统必须显示四人横屏对战桌面
升级游戏必须以横屏模式显示四人座位布局。

#### 场景：座位布局
- **当** 游戏开始
- **那么** 系统必须显示四人座位：上方为对家，左右为对手，下方为玩家自己

#### 场景：手牌显示
- **当** 轮到玩家操作
- **那么** 玩家自己的手牌必须正面显示，其他玩家手牌背面显示

#### 场景：当前出牌显示
- **当** 一轮出牌进行中
- **那么** 系统必须在中央区域显示所有已出的牌

### 需求：系统必须提供叫牌界面
系统必须提供清晰的叫牌操作界面。

#### 场景：叫牌选择
- **当** 叫牌阶段
- **那么** 系统必须显示可叫牌型列表（对子、拖拉机、无将）和"不叫"按钮

#### 场景：花色选择
- **当** 玩家选择叫对子或拖拉机
- **那么** 系统必须显示四种花色供选择

#### 场景：叫牌倒计时
- **当** 轮到玩家叫牌
- **那么** 系统必须显示 35 秒倒计时，超时自动不叫

### 需求：系统必须显示将牌信息
系统必须在游戏过程中持续显示当前将牌信息。

#### 场景：将牌显示
- **当** 叫牌结束
- **那么** 系统必须在界面顶部显示当前级牌和将牌花色

#### 场景：无将显示
- **当** 叫牌结果为无将
- **那么** 系统必须显示"无将"标识

### 需求：系统必须显示计分板
系统必须实时显示两队得分情况。

#### 场景：得分显示
- **当** 游戏进行中
- **那么** 系统必须显示"庄家队 XX 分 / 防守队 XX 分"

#### 场景：得分动画
- **当** 一轮得分产生
- **那么** 系统必须播放得分飞入动画

### 需求：系统必须提供出牌界面
系统必须提供便捷的出牌操作界面。

#### 场景：选牌
- **当** 玩家点击手牌
- **那么** 系统必须高亮选中牌，再次点击取消选中

#### 场景：出牌确认
- **当** 玩家选中牌后点击"出牌"
- **那么** 系统必须验证牌型，合法则出牌，非法则提示错误

#### 场景：自动提示
- **当** 玩家点击"提示"按钮
- **那么** 系统必须高亮推荐出的牌

### 需求：系统必须显示队伍标识
系统必须清晰标识每个玩家所属队伍。

#### 场景：队伍颜色
- **当** 游戏开始
- **那么** 系统必须用不同颜色边框区分两队（如红队/蓝队）

#### 场景：队友高亮
- **当** 显示玩家信息
- **那么** 队友必须显示"队友"标签

### 需求：系统必须提供退出按钮
系统必须在左上角提供统一的退出按钮。

#### 场景：点击退出
- **当** 玩家点击左上角退出按钮
- **那么** 系统必须弹出确认对话框

#### 场景：确认退出
- **当** 玩家确认退出
- **那么** 系统必须执行 `Navigator.pop(context)` 返回上一页

### 需求：系统必须显示游戏结果
游戏结束时必须显示详细结果。

#### 场景：升级结果
- **当** 一局结束
- **那么** 系统必须显示双方得分、升级结果（如"庄家队升 2 级"）

#### 场景：新局开始
- **当** 玩家点击"继续"
- **那么** 系统必须开始新一局，保持队伍和级别信息
