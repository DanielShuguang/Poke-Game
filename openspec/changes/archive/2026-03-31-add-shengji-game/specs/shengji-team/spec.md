# 升级队伍与升级系统规范

## 新增需求

### 需求：系统必须支持四人两两组队
升级游戏必须由 4 名玩家参与，分为两队，对家为队友。

#### 场景：座位分配
- **当** 游戏开始
- **那么** 系统必须按座位顺序分配队伍：座位 0、2 为 A 队，座位 1、3 为 B 队

#### 场景：固定人数
- **当** 玩家尝试加入升级游戏房间
- **那么** 系统必须限制房间人数为 4 人，不允许更多或更少

#### 场景：队伍显示
- **当** 游戏进行中
- **那么** 系统必须清晰显示每个玩家所属队伍（如同色边框）

### 需求：系统必须管理队伍级别
系统必须持久化存储每个队伍的当前级别。

#### 场景：初始级别
- **当** 新队伍首次开始游戏
- **那么** 系统必须将队伍级别初始化为 2

#### 场景：级别递进
- **当** 队伍升级
- **那么** 系统必须按 2→3→4→...→10→J→Q→K→A 顺序递进

#### 场景：A 级后循环
- **当** 队伍完成 A 级胜利
- **那么** 系统必须将级别重置为 2，开始新循环

#### 场景：级别持久化
- **当** 应用关闭后重新打开
- **那么** 系统必须恢复队伍之前的级别进度

### 需求：系统必须管理庄家轮转
系统必须正确管理庄家队伍的轮转。

#### 场景：初始庄家
- **当** 游戏开始
- **那么** 系统必须随机选择一队为初始庄家队

#### 场景：庄家连庄
- **当** 庄家队成功升级（防守队得分 < 80 分）
- **那么** 庄家队必须继续担任庄家

#### 场景：庄家下台
- **当** 防守队得分 ≥ 80 分
- **那么** 庄家队必须下台，防守队成为新庄家

#### 场景：庄家牌权
- **当** 每局开始
- **那么** 庄家必须获得底牌，并首先出牌

### 需求：系统必须处理底牌
系统必须正确处理 8 张底牌的归属和使用。

#### 场景：底牌归属
- **当** 叫牌结束，庄家确定
- **那么** 8 张底牌必须归庄家所有

#### 场景：底牌展示
- **当** 庄家获得底牌
- **那么** 系统必须向所有玩家展示底牌内容

#### 场景：底牌埋分
- **当** 庄家获得底牌后
- **那么** 庄家必须从手牌中选择 8 张牌埋入牌底，这些牌不参与本轮出牌

### 需求：系统必须显示队伍得分
系统必须实时显示两队得分情况。

#### 场景：得分实时更新
- **当** 一轮出牌结束
- **那么** 系统必须立即更新得分方的分数显示

#### 场景：得分差距显示
- **当** 游戏进行中
- **那么** 系统必须显示当前得分差距，如"我方 85 分 vs 对方 115 分"

#### 场景：得分进度条
- **当** 队伍得分接近目标
- **那么** 系统必须以进度条形式显示得分进度
