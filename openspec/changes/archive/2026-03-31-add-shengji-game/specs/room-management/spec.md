# 房间管理增量规范

## 新增需求

### 需求：系统必须支持 shengji 游戏类型标识
房间的 `gameType` 字段必须接受 `shengji` 作为合法值，路由和房间过滤逻辑必须处理该类型。

#### 场景：按游戏类型过滤升级房间
- **当** 玩家在扫描页携带参数 `gameType=shengji`
- **那么** 系统必须仅显示 `gameType` 为 `shengji` 的房间

#### 场景：创建升级房间
- **当** 用户选择游戏类型"升级"
- **那么** 系统必须创建房间，固定人数为 4 人，禁用人数配置

#### 场景：升级房间人数验证
- **当** 用户尝试设置升级房间人数为非 4 人
- **那么** 系统必须拒绝配置并提示"升级游戏固定 4 人"

### 需求：大厅开始游戏路由至升级页面
当升级房间所有玩家准备完成，房主点击"开始游戏"，系统必须正确路由。

#### 场景：导航至升级游戏
- **当** 升级房间 4 名玩家准备完成，房主点击"开始游戏"
- **那么** `RoomLobbyPage._navigateToGame()` 必须创建 `ShengjiNetworkAdapter` 并跳转至 `/shengji`
