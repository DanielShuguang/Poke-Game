# 升级扑克游戏 (Shengji)

## 为什么

项目已上线 5 款扑克游戏，但缺少经典的**四人组队对抗**类游戏。升级（又称"拖拉机"、"80分"）是中国最受欢迎的四人扑克游戏之一，核心特色是**将牌系统**和**持久化升级机制**，能显著丰富游戏类型，吸引更广泛的玩家群体。

## 变更内容

### 新增
- **升级游戏核心引擎**：108张牌（两副牌+大小王）、四人固定、两两组队
- **将牌系统**：动态级牌（2-A循环）+ 叫牌花色 + 无将模式
- **叫牌机制**：对子/拖拉机叫牌，确定庄家和将牌花色
- **出牌规则**：跟牌/杀牌/垫牌、单张/对子/拖拉机牌型
- **计分系统**：分值牌（5/10/K）累计，200分制
- **升级系统**：大光/小光/成功/失败/下台/大反判定
- **AI策略**：叫牌评估、出牌决策（团队配合）
- **局域网联机**：Host/Client 适配器，状态同步

### 修改
- `GameType` 枚举新增 `shengji`（固定4人）
- 路由、首页、大厅、规则页等 7 处接入点

## 功能 (Capabilities)

### 新增功能
- `shengji-engine`: 游戏核心引擎（发牌、叫牌、出牌验证、将牌比较、计分）
- `shengji-team`: 队伍与升级系统（组队、级别、庄家轮转）,,
- `shengji-ai`: AI策略（叫牌决策、出牌决策、团队配合）
- `shengji-network`: 局域网联机适配器（状态同步、超时托管）
- `shengji-ui`: 游戏界面（叫牌界面、四人对战桌面、计分板）

### 修改功能
- `game-selection`: 游戏选择列表新增升级卡片
- `room-management`: 房间创建支持升级（固定4人）

### 技术难点
1. **出牌验证**：跟牌/杀牌/垫牌规则复杂，需完整测试覆盖
2. **将牌比较**：动态级牌 + 花色，大小判断需精确
3. **AI策略**：团队配合决策（对家信号、保护分牌）
4. **拖拉机识别**：连续对子的检测算法

- 复用局域网基础设施（WebSocketManager/Client）
- 复用 AI 策略框架
