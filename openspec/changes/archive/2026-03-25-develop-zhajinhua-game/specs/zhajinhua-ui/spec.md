## 新增需求

### 需求：炸金花游戏界面布局
系统必须提供横屏游戏桌面布局，展示所有玩家、手牌、底池和操作控制面板。

#### 场景：桌面布局
- **当** 炸金花游戏页面加载
- **那么** 系统以横屏模式显示游戏桌面
- **那么** 底部中央显示人类玩家区域
- **那么** 顶部和两侧显示 AI 玩家区域

#### 场景：手牌显示（蒙牌）
- **当** 玩家处于蒙牌状态（`hasPeeked = false`）
- **那么** 系统显示 3 张牌背面（不展示牌面）

#### 场景：手牌显示（看牌后）
- **当** 玩家已看牌（`hasPeeked = true`）
- **那么** 系统翻转展示 3 张牌的牌面

#### 场景：底池显示
- **当** 游戏进行中
- **那么** 桌面中央显示当前底池金额（实时更新）

---

### 需求：下注控制面板
系统必须在玩家轮到时显示操作按钮，非自己回合时禁用所有按钮。

#### 场景：人类玩家回合
- **当** 轮到人类玩家操作
- **那么** 系统启用：看牌（若未看牌）、跟注、加注、弃牌、比牌按钮
- **那么** 系统高亮玩家区域

#### 场景：非玩家回合
- **当** 轮到 AI 玩家操作
- **那么** 系统禁用所有操作按钮
- **那么** 显示"等待 AI 操作"提示

#### 场景：弃牌确认
- **当** 玩家点击"弃牌"按钮
- **那么** 系统弹出确认对话框（防止误操作）
- **那么** 确认后执行弃牌动画

---

### 需求：玩家状态标识
系统必须通过视觉标识区分玩家的状态（蒙牌/看牌/已弃牌/已淘汰）。

#### 场景：蒙牌标识
- **当** 玩家未看牌
- **那么** 玩家区域显示"蒙牌"标签（半透明）

#### 场景：已弃牌标识
- **当** 玩家弃牌
- **那么** 玩家区域变灰并显示"弃牌"标签
- **那么** 玩家的牌背面翻转展示（摊牌效果）

#### 场景：当前回合高亮
- **当** 轮到某玩家操作
- **那么** 该玩家区域外框高亮（金色边框）
