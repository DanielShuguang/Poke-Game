## 新增需求

### 需求：初始化游戏
系统必须支持 2-5 名玩家（1 名真人 + 1-4 名 AI），在开始前配置筹码总量和底注。

#### 场景：开始单机对战
- **当** 用户从主页进入炸金花游戏
- **那么** 系统创建游戏会话，包含 1 名真人玩家和 3 名 AI 玩家（默认），每人持有初始筹码 1000

#### 场景：发牌
- **当** 游戏会话创建
- **那么** 系统随机洗牌（52张，无大小王）并向每名玩家发 3 张牌
- **那么** 每名玩家的牌在未看牌前对自己不可见（蒙牌状态）

---

### 需求：下注轮
系统必须执行轮流下注，每轮每名存活玩家选择跟注、加注或弃牌。

#### 场景：底注收取
- **当** 游戏开始
- **那么** 每名玩家自动缴纳底注（默认 10），计入底池

#### 场景：蒙牌玩家跟注
- **当** 玩家未看牌（`hasPeeked = false`）时选择跟注
- **那么** 系统从其筹码扣除当前底注金额
- **那么** 底池增加对应金额

#### 场景：看牌玩家跟注
- **当** 玩家已看牌（`hasPeeked = true`）时选择跟注
- **那么** 系统从其筹码扣除当前底注金额 × 2
- **那么** 底池增加对应金额

#### 场景：加注
- **当** 玩家选择加注
- **那么** 当前底注翻倍
- **那么** 其他存活玩家必须跟注新底注或弃牌

#### 场景：弃牌
- **当** 玩家选择弃牌
- **那么** 该玩家退出本局游戏，不可再参与下注或比牌
- **那么** 系统显示弃牌动画

#### 场景：只剩一人
- **当** 所有其他玩家弃牌，仅剩一名玩家
- **那么** 该玩家直接赢得底池，游戏结束

---

### 需求：看牌操作
系统必须允许玩家在下注前查看自己的手牌，但看牌后跟注金额加倍。

#### 场景：查看手牌
- **当** 玩家点击"看牌"按钮（处于蒙牌状态）
- **那么** 系统翻开玩家手牌（`hasPeeked = true`）
- **那么** 看牌操作不可撤销

#### 场景：禁止重复看牌
- **当** 玩家已处于看牌状态（`hasPeeked = true`）
- **那么** 系统禁用"看牌"按钮

---

### 需求：比牌
系统必须在两名玩家之间执行比牌，牌小的玩家淘汰。

#### 场景：发起比牌
- **当** 玩家点击"比牌"并选择目标玩家
- **那么** 系统翻开双方手牌
- **那么** 系统评估双方牌型
- **那么** 牌型较小的玩家淘汰出局

#### 场景：比牌平局
- **当** 两名玩家牌型完全相同（点数相同）
- **那么** 发起比牌的玩家淘汰（先手劣势）

---

### 需求：牌型评估
系统必须按照炸金花规则正确评估 3 张牌的牌型并支持大小比较。

#### 场景：豹子识别
- **当** 三张牌点数相同（如 A♠A♥A♦）
- **那么** 系统识别为豹子（`HandRank.threeOfAKind`）

#### 场景：同花顺识别
- **当** 三张牌同花色且连续（如 A♠K♠Q♠）
- **那么** 系统识别为同花顺（`HandRank.straightFlush`）

#### 场景：同花识别
- **当** 三张牌同花色但不连续（如 5♥8♥J♥）
- **那么** 系统识别为同花（`HandRank.flush`）

#### 场景：顺子识别
- **当** 三张牌连续但不同花（如 7♠8♥9♦）
- **那么** 系统识别为顺子（`HandRank.straight`）

#### 场景：顺子特例A-2-3
- **当** 三张牌为 A、2、3（不限花色）
- **那么** 系统识别为顺子，且为最小顺子

#### 场景：对子识别
- **当** 三张牌中恰好两张点数相同
- **那么** 系统识别为对子（`HandRank.pair`）

#### 场景：散牌识别
- **当** 三张牌不满足以上任何牌型
- **那么** 系统识别为散牌（`HandRank.highCard`）

#### 场景：牌型大小比较
- **当** 比较两手牌
- **那么** 豹子 > 同花顺 > 同花 > 顺子 > 对子 > 散牌
- **那么** 同牌型按各自规则比较点数大小

---

### 需求：结算
系统必须在游戏结束时将底池分配给胜者并更新筹码。

#### 场景：胜者赢得底池
- **当** 游戏结束（仅剩一名玩家或比牌决出胜者）
- **那么** 胜者筹码增加底池全额
- **那么** 系统显示结算界面，展示各玩家输赢金额

#### 场景：再来一局
- **当** 结算界面中用户点击"再来一局"
- **那么** 系统重置游戏状态并重新发牌，保留上局结束时的筹码

#### 场景：退出游戏
- **当** 结算界面中用户点击"返回大厅"
- **那么** 系统导航回主页
