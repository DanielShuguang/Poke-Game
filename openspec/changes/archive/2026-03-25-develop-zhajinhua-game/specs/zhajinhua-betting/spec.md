## 新增需求

### 需求：底池管理
系统必须维护底池金额，跟踪每轮下注的累计总额，并在游戏结束时正确分配。

#### 场景：初始底注
- **当** 游戏开始
- **那么** 每名玩家缴纳底注，底池初始金额 = 底注 × 玩家数

#### 场景：跟注入池
- **当** 玩家成功跟注
- **那么** 底池增加玩家实际支付的筹码金额

#### 场景：筹码不足处理
- **当** 玩家筹码不足以支付跟注金额
- **那么** 系统允许玩家以全部剩余筹码跟注（All-in）
- **那么** 底池正常累加，玩家仍可参与最终比牌

#### 场景：胜者分配
- **当** 游戏结束
- **那么** 底池全额分配给胜者
- **那么** 系统更新所有玩家筹码并记录本局输赢

---

### 需求：筹码持久化
系统必须在局间保留玩家的筹码总额（同一次游戏会话内）。

#### 场景：局间筹码保留
- **当** 玩家点击"再来一局"
- **那么** 系统使用上局结束时的筹码开始新局（而非重置为初始值）

#### 场景：筹码耗尽
- **当** 玩家筹码归零
- **那么** 系统提示玩家筹码不足
- **那么** 提供"重新兑换筹码"选项（重置为初始值 1000）
