## 新增需求

### 需求：手牌拖拽滑选
`CardHand` 组件必须支持横向拖拽手势批量选牌：手指按下并划过牌面时，逐张将经过的牌加入拖拽列表；拖拽过程中必须实时计算并高亮"预览牌组"（最小合法组合）；拖拽结束时必须将预览牌组同步为已选状态。

#### 场景：拖拽选中单张
- **当** 玩家手指按下一张牌并立即抬起
- **那么** 该牌进入已选状态（等同于点击）

#### 场景：拖拽跨越多张牌
- **当** 玩家手指按下后横向滑过多张牌
- **那么** 滑过的每张牌依次加入拖拽列表，实时显示预览高亮

#### 场景：拖拽预览高亮
- **当** 拖拽进行中
- **那么** 根据拖拽列表计算最小合法牌组，对应牌以预览高亮样式显示（区别于已选状态）

#### 场景：拖拽结束确认选择
- **当** 玩家抬起手指结束拖拽
- **那么** 若预览牌组非空，则该组牌成为已选状态；预览高亮消除；拖拽中临时禁止的横向滚动恢复正常

#### 场景：拖拽取消
- **当** 拖拽被系统取消（来电、弹窗等）
- **那么** 拖拽状态清除，不改变已选状态

#### 场景：非本玩家回合禁止拖拽
- **当** 当前轮次不是本地玩家
- **那么** 拖拽手势无响应，手牌不可选

### 需求：拖拽与滚动互斥
`CardHand` 组件在拖拽进行时必须禁止 `SingleChildScrollView` 横向滚动，避免选牌手势与滚动冲突。

#### 场景：拖拽期间滚动冻结
- **当** `_isDragging == true`
- **那么** `SingleChildScrollView` 的 `physics` 切换为 `NeverScrollableScrollPhysics`

#### 场景：拖拽结束后滚动恢复
- **当** 拖拽结束或取消（`_isDragging == false`）
- **那么** `SingleChildScrollView` 恢复默认滚动行为
