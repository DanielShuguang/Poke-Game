## 新增需求

### 需求：出牌提示按钮
游戏页面操作区在本地玩家回合时必须显示"提示"按钮（💡图标，琥珀色），点击后自动计算并高亮当前最小合法出法对应的手牌；若无合法出法（无法大过上家且无炸弹），按钮必须禁用。

#### 场景：有合法出法时提示
- **当** 玩家点击"提示"按钮，且存在合法出法
- **那么** `CardHand` 高亮对应手牌的 `hintIndices`；"出牌"按钮自动启用

#### 场景：无合法出法时提示按钮禁用
- **当** 当前手牌无法大过上家且不含炸弹/王炸
- **那么** 提示按钮显示为禁用状态（灰色，`onPressed: null`），点击无响应

#### 场景：提示后点击出牌
- **当** 玩家点击提示后再点击"出牌"
- **那么** 以提示高亮的牌组执行出牌；提示高亮清除

#### 场景：玩家手动改变选牌后提示清除
- **当** 提示高亮状态下玩家点击任意手牌（选中/取消）
- **那么** 提示高亮清除，恢复玩家手动选牌状态

#### 场景：非本玩家回合提示按钮隐藏
- **当** 当前轮次不是本地玩家
- **那么** 提示按钮不显示

### 需求：HintUseCase
必须新增 `HintUseCase`（`domain/paodekai/usecases/hint_usecase.dart`），封装"找当前手牌最小合法出法"逻辑，接受 `List<PdkCard> hand` 和 `PdkPlayedHand? lastPlayedHand`，返回 `List<PdkCard>?`（null 表示无合法出法）。

#### 场景：起手方推荐最小牌
- **当** `lastPlayedHand == null`（新一轮起手）
- **那么** 返回手牌中最小的单张（非炸弹/王炸）

#### 场景：跟牌推荐最小能压过上家的牌
- **当** `lastPlayedHand != null`，手牌中存在同类型且点数更大的组合
- **那么** 返回最小的合法跟牌组合

#### 场景：仅有炸弹可压
- **当** 普通牌型无法压过上家，但手牌含炸弹
- **那么** 返回最小的炸弹

#### 场景：无合法出法
- **当** 手牌任何组合均无法合法压过上家
- **那么** 返回 `null`
