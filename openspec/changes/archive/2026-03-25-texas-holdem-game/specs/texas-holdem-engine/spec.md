## 新增需求

### 需求：游戏初始化与发牌
系统必须在游戏开始时完成洗牌、分配底牌和设置盲注位置，确保每位玩家获得恰好 2 张私有底牌（Hole Cards），公牌区初始为空。

#### 场景：正常发牌
- **当** 房间内玩家人数满足最低要求（≥2人）且游戏开始
- **那么** 系统洗牌后为每位玩家发 2 张底牌，公牌区为空，Dealer Button 指向当前庄家位

#### 场景：盲注自动下注
- **当** 发牌完成后进入 Preflop 阶段
- **那么** Small Blind 位玩家自动扣除小盲注筹码，Big Blind 位玩家自动扣除大盲注筹码，并记录到底池

### 需求：轮次推进（Preflop / Flop / Turn / River）
系统必须按照固定顺序推进游戏阶段：Preflop → Flop（翻3张公牌）→ Turn（翻1张公牌）→ River（翻1张公牌）→ Showdown。每个阶段结束条件为所有活跃玩家的当前轮投注额相等，或仅剩1名活跃玩家。

#### 场景：进入 Flop
- **当** Preflop 投注轮所有玩家 Call 或 Fold 完毕
- **那么** 系统翻开3张公牌，进入新一轮投注，行动从庄家左侧第一位活跃玩家开始

#### 场景：进入 Turn
- **当** Flop 投注轮完成
- **那么** 系统再翻开1张公牌，进入 Turn 投注轮

#### 场景：进入 River
- **当** Turn 投注轮完成
- **那么** 系统再翻开1张公牌，进入 River 投注轮

#### 场景：提前结束（仅剩1名活跃玩家）
- **当** 本轮投注过程中除1名玩家外其余全部 Fold
- **那么** 该玩家直接获得底池，跳过 Showdown，游戏结束

### 需求：Dealer Button 轮转
系统必须在每局游戏结束后将 Dealer Button 顺时针移动到下一位有效玩家，Small Blind 和 Big Blind 位置随之更新。

#### 场景：正常轮转
- **当** 一局游戏结束
- **那么** Dealer Button 移至下一位有序玩家，盲注位置重新计算

#### 场景：玩家离场后轮转
- **当** 某玩家筹码耗尽或离开
- **那么** Dealer Button 跳过该玩家，指向下一位有效玩家

### 需求：游戏状态广播
系统必须在每次状态变化（发牌、翻牌、玩家行动）后将最新 HoldemGameState 广播给所有参与玩家。多人模式下，Host 持有权威状态，客户端禁止直接修改游戏状态。

#### 场景：状态变更广播
- **当** 任意游戏状态发生变化
- **那么** Host 将新状态序列化后广播，所有客户端收到后更新本地视图

#### 场景：客户端行动上报
- **当** 客户端玩家执行行动（Fold / Call / Raise）
- **那么** 客户端仅发送行动指令，Host 验证并执行后再广播结果
