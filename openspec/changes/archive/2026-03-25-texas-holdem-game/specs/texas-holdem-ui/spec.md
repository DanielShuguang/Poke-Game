## 新增需求

### 需求：横屏游戏界面布局
游戏界面必须以横屏模式显示，进入页面时必须强制锁定横屏方向，离开页面时恢复默认方向。界面分为三个区域：顶部公牌区、中部玩家席位区、底部操作区。

#### 场景：进入游戏页面锁定横屏
- **当** 玩家进入德州扑克游戏页面
- **那么** 系统立即将屏幕方向锁定为横屏，竖屏内容不可见

#### 场景：离开页面恢复方向
- **当** 玩家退出德州扑克游戏页面
- **那么** 系统恢复屏幕方向为跟随设备设置

### 需求：公牌区展示
界面顶部中央必须展示公牌区，最多显示5张公牌位置（未翻出的牌位显示牌背）。底池总额必须显示在公牌区下方。

#### 场景：公牌逐张翻出动画
- **当** 系统翻出新公牌
- **那么** 对应位置的牌从牌背翻转为正面，带翻牌动画效果

#### 场景：底池金额更新
- **当** 任意玩家完成下注行动
- **那么** 底池金额实时更新显示

### 需求：玩家席位展示
每个玩家席位必须显示：玩家名称、剩余筹码、当前轮次投注额、底牌（自己正面/对手牌背）、位置标记（Dealer/SB/BB）、行动状态（已弃牌显示灰色）。

#### 场景：当前行动玩家高亮
- **当** 轮到某玩家行动
- **那么** 该玩家席位显示高亮边框，并展示倒计时进度条（默认30秒）

#### 场景：摊牌展示对手底牌
- **当** 游戏进入 Showdown 阶段
- **那么** 所有未弃牌玩家的底牌翻转为正面显示，并标注最优牌型名称

#### 场景：玩家弃牌显示
- **当** 玩家 Fold
- **那么** 该席位底牌消失，席位整体变灰，不再高亮

### 需求：投注操作区
底部操作区必须在轮到玩家行动时显示可用操作按钮（Fold / Check or Call / Raise），并提供加注金额滑动条。系统禁止在非玩家回合显示可交互的操作按钮。

#### 场景：显示操作按钮
- **当** 轮到本机玩家行动
- **那么** 操作区显示 Fold、Call/Check、Raise 按钮，Raise 按钮附带金额输入

#### 场景：Raise 金额调整
- **当** 玩家点击 Raise 按钮
- **那么** 显示滑动条，范围从最小加注额到全部筹码（All-in），快捷选项包含 0.5x底池、1x底池、All-in

#### 场景：非玩家回合隐藏操作
- **当** 当前行动轮不属于本机玩家
- **那么** 操作按钮不可点击或完全隐藏

### 需求：行动超时自动处理
系统必须在玩家超过30秒未操作时自动执行默认行动：若可以 Check 则 Check，否则 Fold。

#### 场景：超时自动行动
- **当** 玩家行动倒计时归零
- **那么** 系统自动执行 Check（无需跟注时）或 Fold（需要跟注时），并广播结果
