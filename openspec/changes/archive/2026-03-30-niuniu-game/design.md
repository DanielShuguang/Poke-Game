## 上下文

项目已有斗地主、德州扑克、炸金花、21点四款扑克游戏，均采用相同的 Clean Architecture + Host/Client 网络适配器模式。牛牛作为第五款游戏，复用相同的房间基础设施（LAN Discovery、RoomLobbyPage、WebSocketManager）。

## 目标 / 非目标

**目标：**
- 实现完整牛牛游戏引擎（判牛、比牌、倍率结算）
- 实现庄家 AI（单机模式下托管其他闲家）
- 实现 Host/Client 联机适配层（复用已有 WebSocket 基础设施）
- 实现横屏游戏 UI（下注→发牌→比牌→结算流程）
- 接入首页、路由和房间大厅

**非目标：**
- 不实现"看牌/不看牌"盲注变种
- 不实现排行榜或积分持久化
- 不实现重连机制

## 决策

### D1：牌型判定算法
从5张牌中枚举 C(5,3)=10 种3张组合，检查点数之和是否能被10整除（J/Q/K/10均计为10，A计为1）。若存在满足条件的组合，则取剩余2张牌点数之和 mod 10 得到"牛X"值（0=牛牛）。

特殊牌型（优先级高于普通牛）：
1. **炸弹**：4张同点数
2. **五小牛**：5张牌全部 ≤5（A/2/3/4/5），点数之和 ≤10
3. **牛牛**：有牛且余2张之和 mod 10 == 0
4. **牛1~牛9**：有牛且余2张之和 mod 10 为 1~9
5. **无牛**：找不到3张之和为10的倍数

倍率：无牛/牛1~牛6 = ×1，牛7 = ×2，牛8 = ×2，牛9 = ×2，牛牛 = ×3，五小牛 = ×5，炸弹 = ×5

### D2：庄家/闲家角色
Host 玩家自动担任**庄家**，其余玩家均为**闲家**；单机模式下人类玩家固定为庄家，AI 控制全部闲家。每局结束后庄家不轮换（由玩家自行决定是否重开，与炸金花保持一致）。

### D3：网络同步
复用炸金花/21点的 Host/Client 模式：
- 下注阶段：Client 发送 `bet` 行动 → Host 广播更新状态
- 发牌后：Host 广播状态（含 `includeAllCards: false`，隐藏他人手牌）
- 翻牌（showdown）阶段：Host 广播含全部手牌的状态
- 35s 超时：闲家超时自动以最小下注确认

### D4：状态阶段
`betting` → `dealing` → `showdown` → `settlement`

不需要逐张发牌交互，发牌后直接进入 showdown 阶段（庄家先翻，闲家依次翻）。

## 风险 / 权衡

- [炸弹判定冲突] 若5张牌既是炸弹又满足五小牛，优先取炸弹 → 明确优先级顺序（炸弹 > 五小牛）
- [大额倍率] ×5 倍率可能快速清空筹码 → 初始筹码设置为 1000，与21点保持一致
- [单机 AI 策略简单] AI 闲家仅做随机下注（50~200），无复杂博弈策略 → 已知限制，后续可扩展
