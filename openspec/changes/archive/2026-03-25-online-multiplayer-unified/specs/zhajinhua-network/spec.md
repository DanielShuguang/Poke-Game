# zhajinhua-network 规范

## 新增需求

### 需求：炸金花联机网络适配器
系统必须提供 ZhjNetworkAdapter，负责炸金花联机对战的消息路由与状态同步，采用 Host/Client 模式与现有 WebSocket 基础设施集成。

#### 场景：Host 接收客户端行动
- **当** Host 收到来自客户端的炸金花行动消息（`type: "zhj_action"`）
- **那么** 系统必须验证行动发起方是否为当前回合玩家
- **那么** 系统必须执行对应行动（跟注/加注/弃牌/看牌/比牌）并将完整游戏状态广播给所有玩家

#### 场景：Client 接收状态同步
- **当** Client 收到来自 Host 的状态广播消息（`type: "zhj_state_sync"`）
- **那么** 系统必须用广播数据更新本地游戏状态
- **那么** 系统必须屏蔽本地玩家以外的手牌数据（不展示其他玩家牌面，除非该玩家已翻牌）

#### 场景：行动超时处理
- **当** 玩家收到轮次通知后 35 秒内未发出行动
- **那么** Host 必须代该玩家执行弃牌并广播新状态

### 需求：炸金花联机状态序列化
系统必须支持 ZhjGameState 的完整序列化与反序列化，以便通过 WebSocket 传输。

#### 场景：序列化游戏状态
- **当** Host 执行行动后需要广播状态
- **那么** 系统必须将 ZhjGameState 序列化为 JSON，包含：底池、当前底注、当前玩家索引、所有玩家的筹码/弃牌状态/看牌状态，以及 Host 端可见的手牌（用于胜负判定）

#### 场景：反序列化游戏状态（客户端）
- **当** Client 接收到 JSON 状态数据
- **那么** 系统必须将 JSON 还原为本地 ZhjGameState，对非本地玩家的手牌字段置为空列表

### 需求：炸金花联机 UI 分支
系统必须在炸金花游戏页面支持联机模式，区别于单机模式的交互行为。

#### 场景：联机模式禁用非自己回合操作
- **当** 炸金花页面以联机模式（`isOnline: true`）启动且当前回合不是本地玩家
- **那么** 系统必须禁用所有操作按钮并显示"等待其他玩家操作"提示

#### 场景：Client 操作发送网络行动
- **当** 联机模式下本地玩家点击跟注/加注/弃牌/看牌/比牌按钮
- **那么** 系统必须通过 ZhjNetworkAdapter.sendAction 发送行动至 Host，而非直接修改本地状态

#### 场景：Host 模式下直接执行行动
- **当** 联机模式下 Host 玩家点击操作按钮
- **那么** 系统必须直接执行本地行动并通过 ZhjNetworkAdapter 广播新状态
