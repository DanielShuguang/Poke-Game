# room-management 增量规范

## 修改需求

### 需求：用户可以创建游戏房间
系统必须允许用户创建新房间，设置房间名称、游戏类型、最大人数等参数。创建房间时，德州扑克和炸金花必须为可选游戏类型，不再仅限斗地主。

#### 场景：创建斗地主房间
- **当** 用户选择"创建房间"，选择游戏类型"斗地主"，输入房间名称"欢乐对战"
- **那么** 系统必须创建房间，固定人数为 3 人，禁用人数配置输入，并自动成为房主和玩家

#### 场景：创建德州扑克房间
- **当** 用户选择游戏类型"德州扑克"，通过 Slider 设置最大人数（2-9 人范围内）
- **那么** 系统必须创建房间，允许所选人数加入，房主自动成为玩家

#### 场景：创建炸金花房间
- **当** 用户选择游戏类型"炸金花"，通过 Slider 设置最大人数（2-5 人范围内）
- **那么** 系统必须创建房间，允许所选人数加入，房主自动成为玩家

#### 场景：切换游戏类型重置人数
- **当** 房主在创建房间时切换游戏类型（如从德州扑克切换为炸金花）
- **那么** 系统必须将最大人数重置为新游戏类型的最大值，并更新 Slider 范围

#### 场景：人数配置 UI 随游戏类型变化
- **当** 游戏类型为斗地主
- **那么** 系统必须显示"固定 3 人"文本，隐藏人数 Slider
- **当** 游戏类型为德州扑克或炸金花
- **那么** 系统必须显示人数 Slider，范围来自游戏类型的 minPlayerCount / maxPlayerCount
