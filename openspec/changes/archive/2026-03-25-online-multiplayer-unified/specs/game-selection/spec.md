# game-selection 增量规范

## 新增需求

### 需求：游戏卡片提供联机对战入口
系统必须在德州扑克和炸金花游戏卡片上提供独立的"联机对战"入口，允许玩家直接进入联机房间流程。

#### 场景：德州扑克卡片显示联机入口
- **当** 玩家在主页游戏选择区浏览
- **那么** 德州扑克游戏卡片必须显示"联机对战"按钮，点击后导航至房间扫描页并过滤显示德州扑克房间

#### 场景：炸金花卡片显示联机入口
- **当** 玩家在主页游戏选择区浏览
- **那么** 炸金花游戏卡片必须显示"联机对战"按钮，点击后导航至房间扫描页并过滤显示炸金花房间

#### 场景：单机与联机入口并存
- **当** 玩家在主页游戏选择区浏览
- **那么** 点击游戏卡片主体必须进入单机 AI 对战模式（现有行为不变）
- **那么** 点击"联机对战"按钮必须进入联机房间流程（新增行为）

## 修改需求

### 需求：系统必须验证人数配置
系统必须根据游戏规则验证并限制房间人数配置。

#### 场景：斗地主固定人数
- **当** 游戏类型为"斗地主"
- **那么** 系统必须强制房间人数为 3 人，忽略房主的配置

#### 场景：德州扑克人数限制
- **当** 游戏类型为"德州扑克"且房主设置人数为 10 人
- **那么** 系统必须拒绝配置并提示"德州扑克最多支持 9 人"

#### 场景：炸金花人数限制
- **当** 游戏类型为"炸金花"且房主设置人数超过 5 人
- **那么** 系统必须拒绝配置并提示"炸金花最多支持 5 人"
