## 新增需求

### 需求：联机行动消息格式
所有掼蛋联机行动必须通过 JSON 消息传输，消息必须包含 `type` 字段；Host 收到消息后必须验证合法性，非法消息禁止执行。

#### 场景：出牌消息合法
- **当** Client 发送 `{"type":"playCards","cards":["AH","AS","KH","KS","QH"]}`
- **那么** Host 验证牌型合法后执行出牌，广播新状态

#### 场景：pass 消息
- **当** Client 发送 `{"type":"pass"}`
- **那么** Host 执行 pass，广播新状态

#### 场景：非法出牌被拒绝
- **当** Client 发送的 `cards` 不构成合法牌型
- **那么** Host 忽略该消息，不改变游戏状态

---

### 需求：贡牌阶段消息同步
贡牌与还贡行动必须通过独立消息类型传输；Host 必须顺序处理贡牌消息，防止并发冲突。

#### 场景：进贡消息
- **当** Client 发送 `{"type":"tribute","card":"AH"}`
- **那么** Host 验证该牌是发送方手中最大单张后执行，广播中间态（含贡牌归属）

#### 场景：还贡消息
- **当** Client 发送 `{"type":"returnTribute","card":"3D"}`
- **那么** Host 验证合法后执行还贡，若所有贡牌完成则广播出牌阶段状态

#### 场景：并发贡牌顺序处理
- **当** 两个 Client 同时发送进贡消息
- **那么** Host 按消息到达顺序依次处理，不产生状态冲突

---

### 需求：手牌隐藏
Host 广播状态时，非本地玩家的手牌内容必须隐藏；手牌仅向对应玩家本人可见。

#### 场景：Client 收到状态，他人手牌不可见
- **当** Host 广播 `toJson(includeAllCards: false)` 的状态
- **那么** Client 收到的状态中，非本人玩家的手牌列表为空或仅含张数信息

#### 场景：showdown 阶段手牌全部可见
- **当** 一局结束进入结算
- **那么** Host 广播 `toJson(includeAllCards: true)` 的状态，所有手牌对所有 Client 可见

---

### 需求：35 秒超时托管
Host 必须为每个玩家行动计时；若 35 秒内未收到合法行动消息，Host 必须代为执行最小操作。

#### 场景：出牌阶段超时
- **当** 当前玩家 35 秒内未出牌或 pass
- **那么** Host 代为执行 pass（或出最小合法牌若为首出）

#### 场景：贡牌阶段超时
- **当** 应贡牌玩家 35 秒内未提交贡牌
- **那么** Host 代为自动选择该玩家手牌中最大单张进贡

#### 场景：收到合法行动后取消超时
- **当** Host 在超时前收到合法行动消息
- **那么** Host 取消当前计时器，执行该行动
