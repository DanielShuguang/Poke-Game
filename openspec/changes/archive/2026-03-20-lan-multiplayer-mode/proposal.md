# 局域网多人游戏模式

## 为什么

当前应用仅支持单机人机对战，缺乏多人实时对战能力。局域网多人模式可以让玩家在同一 WiFi 环境下与朋友实时对战，无需互联网连接和账号系统，降低使用门槛，提升社交娱乐体验。

## 变更内容

引入局域网多人游戏功能，支持：

- **房间创建与发现**：设备可在局域网内创建或发现游戏房间
- **游戏类型选择**：房主可从游戏列表中选择对战类型（斗地主、德州扑克等）
- **人数自定义**：房主可设置参与人数（受游戏规则限制，如斗地主固定 3 人）
- **实时同步**：游戏状态通过局域网实时同步至所有玩家
- **跨平台支持**：Android、iOS、Windows、macOS 等设备可互相连接

## 功能 (Capabilities)

### 新增功能

- `lan-discovery`: 局域网设备与房间发现机制，支持自动扫描和手动输入 IP
- `room-management`: 房间创建、配置、加入、退出和销毁的全生命周期管理
- `game-selection`: 游戏类型选择与规则配置，支持人数验证和游戏特定设置
- `player-management`: 玩家身份、状态（准备、游戏中、离线）、座位管理
- `network-sync`: 游戏状态同步机制，包括发牌、出牌、叫地主等核心游戏动作
- `chat-system`: 房间内文字聊天与预设表情系统（可选）

### 修改功能

- `doudizhu-game`: 扩展现有斗地主游戏以支持多人局域网对战模式，调整 AI 逻辑为可选

## 影响

### 代码架构

- **新增模块**：
  - `lib/core/network/` - 网络通信层（Socket/HTTP 服务发现）
  - `lib/domain/lan/` - 局域网业务层（实体、用例、仓库接口）
  - `lib/presentation/pages/room/` - 房间相关 UI 页面

- **修改模块**：
  - `lib/domain/doudizhu/` - 扩展游戏状态管理以支持网络同步
  - `lib/core/router/` - 新增房间创建、发现、等待页面路由

### 技术依赖

- **新增依赖**：
  - 网络通信：`socket_io_client` 或 `dart:io` 原生 Socket
  - 服务发现：`nsd` (Android/iOS) 或 `multicast_dns` (跨平台)
  - 状态码：`freezed` 用于不可变消息模型

- **平台权限**：
  - Android: `ACCESS_WIFI_STATE`, `CHANGE_WIFI_MULTICAST_STATE`
  - iOS: Bonjour 服务权限配置

### 现有功能影响

- 单机人机模式保持不变，作为默认游戏模式
- 局域网模式作为可选功能入口，不影响现有用户流程
- 游戏逻辑层抽象化，支持本地和远程玩家统一接口
