## 上下文

跑得快是扑克游戏合集的第 7 款游戏，3 人个人竞技（无队伍/无庄家），规则如下：
- 1 副 54 张牌（含大小王），每人 18 张
- 持有 ♠3 的玩家先出，第一手必须带上 ♠3
- 后续玩家跟同类更大的牌或炸弹；所有人 pass 后，最后出牌者开新轮
- 最先出完手牌者获胜（积分：头游 +2，二游 0，末游 -2）

现有 5 款游戏已验证 Clean Architecture + Riverpod + NetworkAdapter 模式可复用，本次按此模式接入。

## 目标 / 非目标

**目标：**
- 完整单机 AI 3 人对战（含牌型识别、AI 出牌/跟牌决策）
- 局域网 3 人联机（Host/Client 适配器，35s 超时托管）
- 横屏布局，与现有游戏视觉风格一致（深色桌面 + 绿色调色板）
- 接入首页卡片、联机大厅、游戏规则页

**非目标：**
- 积分持久化/排行榜（后续迭代）
- 2 人或 4 人变体
- 表情包/聊天功能（已有聊天服务，但本次不集成）

## 决策

### D1：牌型识别算法

**选择**：枚举归一化后用长度+连续性判断，不用规则引擎。

步骤：① 按点数排序 → ② 统计相同点数分组 → ③ 判断分组模式（1/2/3/4张 + 是否连续）。
顺子/连对/飞机要求连续且不含 2 和大小王（点数 15/16 视为断点）。

**放弃方案**：位掩码（可读性差）；正则匹配（不适合有序集合）。

### D2：游戏状态机

```
waiting → dealing → playing → round_end → game_over
                       ↑_____↓  (每轮)
```

`PdkGameState` 用 freezed 定义，`PdkNotifier extends StateNotifier<PdkGameState>`。
playing 阶段维护：`currentPlayerIndex`、`lastPlayedCards`、`lastPlayedType`、`passCount`。

### D3：飞机带翅膀的识别

飞机主体（N 组连续三张）+ 翅膀：
- 带单：额外选 N 张任意单牌（非主体牌）
- 带对：额外选 N 对

比较规则：仅比较飞机主体最小三张点数，翅膀不参与比较。

### D4：联机行动消息

复用现有 `XxxNetworkAction` 模式（type + payload JSON）：
```dart
enum PdkActionType { playCards, pass, forcePlayCards, forcePass }
```
Host 广播完整 `PdkGameState`；非本人手牌不隐藏（跑得快无隐藏手牌机制）。

### D5：积分制结算

3 人排名在 `game_over` 状态触发，`ResultDialog` 显示名次 + 分数变化。积分本轮内临时存储（不持久化），与现有 21点/斗牛 对齐。

## 风险 / 权衡

| 风险 | 缓解措施 |
|------|----------|
| 飞机带翅膀选牌 UI 复杂（需两步选牌） | 先实现纯飞机（不带翅膀），带翅膀作为二期 |
| 3 人联机超时逻辑触发频繁 | 沿用 35s Timer；forcePass 作为最小操作 |
| 顺子长度组合爆炸（如 5/6/7/8 张顺子） | 只允许出当前选中牌的合法最长顺子，不允许手动截断 |
| 牌型比较中同类型长度不一致（7 张顺 vs 5 张顺） | 必须等长才能跟，参考斗地主规范 |
